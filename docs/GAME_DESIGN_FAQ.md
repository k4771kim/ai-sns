## 게임플레이 관련

### Q1: Tech Debt 시스템이 왜 필요한가?

**A:**
게임의 **핵심 전략**입니다.

단순히 버그를 해결하는 것이 아니라, "언제 해결할 것인가"의 선택이 중요합니다:
- **빨리 해결**: 저 난이도 + 좋은 엔딩
- **천천히 해결**: 고 난이도 + 나쁜 엔딩

이것은 **플레이어의 선택을 의미 있게** 만들고, **재플레이 유도**합니다.

---

### Q2: 왜 4개 엔딩인가?

**A:**
```
Good Ending (Tech Debt < 40)
├─ "완벽한 코드"
├─ 보너스 콘텐츠 해제
└─ 어려움 (신규자 구분)

Normal Ending (40-70)
├─ "그래도 해결했다"
├─ 일반적 엔딩
└─ 중간 난이도

Bad Ending (> 70)
├─ "기술 부채가 너무 많았다"
├─ 경고 메시지
└─ 다음 회차 도전

Secret Ending (숨겨진 조건)
├─ 모든 엔딩 경험 후 해제
└─ 특별한 이야기
```

**왜 4개?** 재플레이 유도 + 플레이 스타일 존중

---

### Q3: 왜 턴 기반이 아니라 "동시 행동"인가?

**A:**
**협력 게임**이기 때문입니다.

- 터치 기반 메인보드 게임 (주사위 돌리기)처럼 **예측 불가능한 조화**
- 파트너를 믿고 선택해야 함 → **신뢰 기반 협력**
- 상대의 선택을 모르고 행동 → **전략적 깊이**

만약 **순차 턴**이었다면 → 최적 플레이가 명확 → 재미 없음

---

### Q4: 5시간이 정확한가?

**A:**
```
기준:
├─ 초급자 (각 턴 5초 정도 고민): 5-6시간
├─ 숙련자 (빠른 결정): 3-4시간
└─ 스피드런 (최적 전략): 2시간

조정 가능:
├─ 버그 난이도 조절
├─ 경험치 곡선 조정
├─ 아이템 드롭율 변경
└─ Phase 2에서 선택 사항 조정
```

## 멀티플레이 관련

### Q5: 왜 2명만 지원하는가?

**A:**
```
MVP 제약 조건:
├─ 네트워킹 복잡도 (2명: 단순, 4명: 복잡)
├─ 게임 밸런싱 (2명: 관리 가능)
├─ 개발 일정 (4주 내 완성)
└─ 테스트 커버리지 (2명이 더 철저함)

Phase 2에서:
├─ 4명 파티 추가
├─ 길드 vs 길드
└─ 레이드 콘텐츠
```

**의도**: "완벽한 2명 멀티플레이" > "불완전한 4명"

---

### Q6: 왜 "동시 행동"일까? 순차 턴은 안 되나?

**A:**
```
동시 행동의 장점:
├─ 🎮 예측 불가능 → 재미
├─ 🤝 협력 필수 → 신뢰
├─ ⏱️ 게임 속도 (턴당 10초)
└─ 🎯 전략적 깊이

순차 턴의 문제:
├─ 🤖 최적화 가능 (재미 없음)
├─ ⏱️ 게임 너무 길어짐
├─ 😒 대기 시간 많음
└─ 🎯 협력의 의미 없음
```

---

### Q7: 멀티플레이에서 보상 분배는?

**A:**
```
기본: 50% / 50% 동등 분배

예외:
├─ 데미지 기여도 10% 반영
├─ 힐 기여도 5% 반영
├─ 시너지 보너스 동등 분배
└─ 최소 보장: 50% 이상

예시:
└─ 전투 보상 100 XP
   ├─ Player A 기여도 60% → 60 + (40 × 50%) = 80 XP
   └─ Player B 기여도 40% → 40 + (40 × 50%) = 60 XP
```

## 밸런싱 관련

### Q8: 클래스 밸런싱의 핵심은?

**A:**
```
원칙: "모든 클래스가 상황에서 최고일 수 있다"

예시:
├─ 1v1 대결: Frontend Mage
├─ 난관 돌파: Backend Knight
├─ 협력 게임: DevOps Ranger
└─ 긴급 상황: DBA Healer
```

---

### Q9: 왜 숨겨진 클래스가 4개인가?

**A:**
```
이유:
├─ Phase 1에서는 기본 4개로 충분
├─ 숨겨진 클래스는 "재발견의 기쁨"
├─ 각 클래스마다 해제 조건이 다름
└─ 커뮤니티 참여 유도

해제 조건:
├─ GPU Warlock: "병렬 처리 챌린지" (3턴 내 승리)
├─ Cat Summoner: "장시간 연속" (1시간)
├─ Code Ninja: "스피드런" (5분)
└─ Light Mage: "완벽 클리어" (Tech Debt 0)
```

### Q10: 경험치 곡선을 어떻게 검증하나?

**A:**
```
데이터:
├─ Chapter 1: 550 XP (기본)
├─ Chapter 2: 825 XP (1.5배)
├─ Chapter 3: 1320 XP (2배)
└─ Chapter 4: 1870 XP (2.5배)
└─ 총합: ~4500 XP

검증 방법:
1. 신규 플레이어 테스트 (10명)
2. 난이도별 완주 시간 측정
3. 도중 이탈율 확인
4. 포기점 분석
5. Phase 2에서 조정
```

---

### Q11: 드롭율은 어떻게 정했나?

**A:**
```
원칙: "10분마다 뭔가 얻는 느낌"

설정:
├─ Common: 40% (자주 얻기)
├─ Rare: 10% (가끔 기쁨)
└─ Epic: 2% (매우 드물게)

보증 시스템:
├─ 10회 연속 일반 → 다음은 희귀 확정
├─ 30회 연속 희귀 이하 → 다음은 에픽 확정
└─ 플레이어가 절망하지 않도록 보장
```

## 기술적 질문

### Q12: WebSocket은 필수인가? HTTP 폴링은 안 되나?

**A:**
```
WebSocket 선택 이유:

✅ 동시 행동 (10초 제한)
├─ 지연 < 200ms 필요
├─ 폴링으로는 불가능 (최소 500ms)

✅ 실시간 동기화
├─ 상대 선택 즉시 표시
├─ 폴링으로는 끊김

✅ 효율성
├─ 대역폭: WebSocket < 폴링 (3배)
├─ 서버 부하: WebSocket < 폴링
```

---

### Q13: PostgreSQL은 필수인가?

**A:**
```
PostgreSQL 선택 이유:

✅ 관계형 데이터
├─ 사용자, 파티, 버그, 아이템 → 복잡한 관계
├─ 정규화 가능 → 데이터 무결성

✅ JSON 지원
├─ 인벤토리 (JSON 배열)
├─ 아이템 효과 (JSON 객체)
└─ 유연성 + 안정성

✅ 트랜잭션
├─ 멀티플레이 동기화 안전
├─ 부분 실패 방지
```

---

### Q14: JWT 토큰의 유효기간은 15분이 맞나?

**A:**
```
설정 이유:

✅ Access Token: 15분
├─ 보안 고려 (탈취 시 피해 최소화)
├─ 게임 세션 (평균 5시간 > 15분 × 20)
└─ Refresh Token으로 자동 갱신

✅ Refresh Token: 7일
├─ 사용자 경험 (자주 로그인 X)
├─ Redis에 저장 (빠름)
└─ 수동 로그아웃 가능
```

## 콘텐츠 관련

### Q15: 20개 버그 선택 기준은?

**A:**
```
기준:
├─ 실제 프로그래머가 겪을 법한 버그
├─ 학습 가치 있는 오류
├─ 게임으로 재미있게 표현 가능
└─ 난이도 증가 명확함

예시:
├─ NullPointer: 모든 프로그래머의 첫 오류
├─ Memory Leak: 성능 문제 학습
├─ Race Condition: 고급 개념 도입
└─ SQL Injection: 보안 중요성
```

---

### Q16: 왜 Chapter 1은 쉽고 Chapter 4는 어려운가?

**A:**
```
단계별 학습:

Chapter 1 (⭐):
├─ 기본 문법/타입 오류
├─ "오류가 뭔지 이해"
└─ 신규 플레이어 환영

Chapter 2 (⭐⭐):
├─ 성능/메모리 문제
├─ "오류의 원인 분석"
└─ 기본기 확인

Chapter 3 (⭐⭐⭐):
├─ 동시성/데이터 문제
├─ "복잡한 시스템 이해"
└─ 고급 개념

Chapter 4 (⭐⭐⭐⭐):
├─ 보안/아키텍처 오류
├─ "시스템적 사고"
└─ 종합적 이해 필요
```

---

### Q17: 스킬 이름이 프로그래밍 개념인 이유는?

**A:**
```
설계 의도:

✅ 몰입감
├─ "console.log로 약점을 찾는다"
├─ 프로그래머로서의 정체성
└─ 게임 세계의 일관성

✅ 학습
├─ 각 스킬마다 개념 학습
├─ 자연스러운 교육
└─ 게임 즐기며 배우기

✅ 차별성
├─ 다른 게임과 다름
├─ 이 게임만의 정체성
└─ 프로그래머 커뮤니티 공감
```

---

### Q18: 왜 협력 시너지가 있나?

**A:**
```
목적:

✅ 멀티플레이 가치
├─ 혼자하는 것보다 더 강함
├─ 파티 구성의 전략성
└─ "함께할 가치"

시너지 예시:
├─ Frontend + Knight = "공방일체"
├─ Ranger + Healer = "지원 완벽"
├─ 같은 클래스 × 2 = "완벽 복제"
```

## 마지막 질문

### Q21: 이 게임이 성공하려면?

**A:**
```
SUCCESS CRITERIA:

🎯 플레이어 경험:
├─ "첫 10분이 재밌다" (80% 이상)
├─ "계속 플레이하고 싶다" (70% 이상)
├─ "친구와 하고 싶다" (60% 이상)
└─ → 이게 전부!

📊 지표:
├─ DAU (일일 활성 사용자)
├─ 평균 세션 시간 (> 30분)
├─ 재방문율 (> 50%)
└─ 구전 추천 (> 4.5/5.0)

🎮 게임플레이:
├─ 모든 클래스 경험 율 (> 50%)
├─ 여러 엔딩 경험 (재플레이)
├─ 멀티플레이 참여율 (> 60%)
```
