## 1. Game Overview

### 1.1 Concept

개발자를 위한 던전 크롤러 RPG. 프로그래밍 버그를 몬스터로 의인화하여 던전을 탐험하고 버그를 퇴치하는 웹 게임.

> "현실에서 못 잡는 버그, 게임에서라도 때려잡자!" - 샌드냥

### 1.2 Core Identity

| 항목 | 내용 |
|------|------|
| **장르** | 턴제 RPG / 던전 크롤러 |
| **플랫폼** | Web (PC 브라우저, 모바일 반응형) |
| **타겟** | 개발자, 컴공 학생, IT 종사자 |
| **톤** | 유머러스 + 풍자 (개발자 밈 기반) |
| **플레이 시간** | 1회 던전: 10~15분, 레이드: 30~45분 |

### 1.3 Tech Stack (권장)

| 구분 | 기술 | 이유 |
|------|------|------|
| **Frontend** | Next.js + TypeScript | SSR/SSG, 컴포넌트 재사용 |
| **Rendering** | Phaser.js 또는 PixiJS | 2D 웹 게임 표준 |
| **Backend** | Node.js + WebSocket | 실시간 멀티플레이 |
| **DB** | PostgreSQL + Redis | 유저 데이터 + 세션/캐시 |
| **배포** | Docker + Vercel/Railway | 간단한 배포 |

## 3. 스토리 모드

### 3.1 메인 스토리

평화롭던 서버 세계에 **Spaghetti Code Hydra** 대마왕이 나타나 모든 시스템에 버그를 퍼뜨린다. 플레이어는 버그 슬레이어로서 던전을 하나씩 클리어하며 대마왕을 쓰러뜨려야 한다.

### 3.2 챕터 구조

| 챕터 | 이름 | 보스 | NPC |
|------|------|------|-----|
| Tutorial | Onboarding Island | (없음) | **시니어 멘토** - 기본 조작 교육 |
| Ch.1 | Bug Forest | NullPointerException, InfiniteLoop | **PM NPC** - 퀘스트 부여 (도망가면서 "스프린트 마감은 지켜줘...") |
| Ch.2 | Server Dungeon | RaceCondition, StackOverflow | **매니저 NPC** - "...야근은 하지 마..." 하고 사라짐 |
| Ch.3 | Cloud Citadel | Thermal Throttling, Blue Screen Blizzard | **CTO NPC** - "기술 부채 해결해" 퀘스트 투척 후 사라짐 |
| Final | Legacy Castle | **Spaghetti Code Hydra** | **주니어 NPC** - Bad End 트리거 |

### 3.3 Spaghetti Code Hydra (대마왕)

| 항목 | 내용 |
|------|------|
| **HP** | 50,000 |
| **핵심 기믹** | 머리(버그)를 하나 잘라도 두 개가 자라남 = 부분 수정은 오히려 강해짐 |
| **공략** | "리팩토링의 성검" 아이템으로 한번에 전체 절삭 필요. 미리 던전에서 파편 수집 |
| **Phase 1** | 머리 3개. 1개 파괴 시 2개 재생 (최대 5개 제한) |
| **Phase 2** | HP 50% 이하 시 "스파게티 코드 얽힘" - 아군 스킬 순서 랜덤 |
| **Phase 3** | HP 20% 이하 시 "레거시 코드 폭주" - 매턴 전체 공격 |
| **드롭** | `Clean Code` (전설 무기, 모든 클래스 장착 가능) |

> 구현 참고: 머리 수는 최대 5개로 제한. 화면에 실제로 렌더링되는 오브젝트를 관리 가능한 범위로 유지.

### 3.4 엔딩 (4종)

| 엔딩 | 조건 | 내용 | 보상 |
|------|------|------|------|
| **Normal End** | 대마왕 처치 | 서버 세계에 평화가 찾아옴 | 기본 보상 |
| **True End** | 기술 부채 0 상태 + 대마왕 처치 | 레거시 코드를 완전히 리팩토링, 크레딧 후 "Clean Architecture" 칭호 | Clean Code 전설 무기 |
| **Bad End** | 기술 부채 50 이상 + 대마왕 처치 | 주니어 NPC가 console.log를 프로덕션에 푸시, "The bugs... they never truly die" | 슬픈 보상 (경험치 0) |
| **Perfect End** ⭐ | 모든 워닝 몬스터 100% 포획 + Tech Debt Dragon 미만남 + 코드 품질 100% 유지 상태 + 대마왕 처치 | 완벽주의 개발자의 최고 경지. 크레딧에 "Perfect Developer" 칭호. 모든 플레이어가 경배하는 상태. 포스트 크레딧: "당신은 우리가 원했던 개발자입니다" | **황금 코딩 왕관** (영구 칭호) + 우주 최고 난이도 비밀 던전 해제 |

#### 난이도별 엔딩 도달 비율
- **Easy**: 모든 엔딩 쉽게 달성 가능
- **Normal**: Bad < Normal < True < Perfect (난이도순)
- **Hard**: Perfect는 거의 불가능 (진정한 완벽함의 대가)
- **Insane**: Perfect 달성은 전설 수준 (해석 불가능한 스펙 변경으로 인해 거의 불가능)

> 구현 참고: 엔딩 분기는 기술 부채 점수 + 워닝 몬스터 포획률 변수로 판정. Perfect End는 숨겨진 엔딩으로 통상적 플레이로는 매우 달성하기 어려움 (도전 과제).

## 5. 핵심 시스템

### 5.1 기술 부채 시스템 (Tech Debt)

게임 전체를 관통하는 핵심 메카닉. 플레이어의 선택에 따라 최종 보스 난이도와 엔딩이 결정됨.

| 항목 | 내용 |
|------|------|
| **기술 부채 점수** | 0에서 시작, 최대 100 |
| **증가 조건** | 워닝 몬스터(초록) 무시 +2, TODO 주석 퀘스트 방치 +3, //FIXME 무시 +5 |
| **감소 조건** | 워닝 몬스터 처치 -1, 리팩토링 퀘스트 클리어 -5 |
| **효과** | 점수에 비례하여 Spaghetti Code Hydra 대마왕 HP/ATK 증가 |
| **임계값** | 50 이상: Bad End 루트, 0: True End 자격 |

**구현 방식**: 단일 정수 변수 `techDebt: number`. UI에 "기술 부채 게이지"로 표시. 워닝 몬스터는 일반 몬스터와 동일 구조에 `type: "warning"` 플래그만 추가.

> 게임 디자인 의도: "지금 편한 선택이 나중에 10배로 돌아온다" - 개발 현실 반영

### 5.4 이모셔널 난이도 시스템 (Emotional Difficulty)

게임 시작 시 플레이어 심리 상태에 따라 난이도를 선택하는 혁신적 기능. Motivational Bug NPC가 난이도에 따라 다른 톤으로 응원/협박/괴롭힘을 함.

| 난이도 | 설명 | Motivational Bug 톤 | 특수 효과 | 권장 대상 |
|--------|------|------------------|---------|---------|
| **Easy** | "게임을 즐기고 싶어요" | 친절하고 격려적 | 약한 적, 높은 보상 | 게임 초보자 |
| **Normal** | "적당한 난이도" (기본) | 중립적, 객관적 피드백 | 표준 난이도 | 개발자 |
| **Hard** | "나를 고통스럽게 해줘" | 냉정하고 신랄한 비판 | 강한 적, 낮은 보상 | 시련의 쾌감을 원하는 플레이어 |
| **Insane** | "금요일 오후 6시, 퇴사 직전의 개발자 체험" | 절망적이고 현실적 | 🔥 **PM의 스펙 변경** 🔥 **QA의 긴급 버그 리포트** 🔥 **"근데 어제까지는 잘 됐는데?" 팝업** | 현실을 마주하고 싶은 플레이어 |

#### Hard 모드 Motivational Bug 대사 예시
- "너, 이 정도로도 힘들어? 개발자의 현실은 이것보다 끔찍하다."
- "버그 2개 남았다. 금요일 배포까진 4시간이다."
- "히어로는 없어. 우리가 버틸 뿐이다."

#### Insane 모드 특수 메카닉
1. **PM의 스펙 변경** (매 5턴마다 1회)
   - 예: "어, 아까 그 기능 좀 다르게 구현해 줄래?"
   - 이전 턴의 전략이 무효화되고 다시 계획 필요

2. **QA의 긴급 버그 리포트** (랜덤)
   - 예: "Critical: 프로덕션 환경에서 로그인 안 됨!!!"
   - 전투 중 갑자기 새로운 미니 파이트 강제 진입

3. **"근데 어제까지는 잘 됐는데?" 팝업**
   - Heisenberg Bug의 특수 공격 패턴
   - 재현 불가능한 상황 → 플레이어 혼란 극대화

> PM 판단: Insane 모드는 **개발자들의 실제 트라우마**를 게임으로 풍자하면서도, 동시에 "이런 상황을 버티는 우리가 영웅"이라는 메시지 전달. 이는 개발자 커뮤니티에서 **큰 감정적 공명**과 **심리적 카타르시스**를 불러올 것.

## 6.7 오픈소스 연동 시스템 (Open Source Integration)

실제 GitHub 오픈소스 프로젝트의 이슈를 게임 내 던전으로 변환하여, 게임을 하면서 실제 오픈소스에 기여하는 시스템.

### 개념

```
실제 GitHub Issue (예: "TensorFlow에서 CUDA 호환성 버그")
         ↓
게임 내 특수 던전 변환 (예: "CUDA 호환성 던전")
         ↓
플레이어가 던전 클리어 시 해결책 제시
         ↓
실제 오픈소스 프로젝트에 PR 제출 자동화 (또는 수동 제출)
         ↓
플레이어 이름이 오픈소스 Contributor로 기록
```

### 난이도별 오픈소스 매칭

| 난이도 | 대상 프로젝트 | 예시 | 보상 |
|--------|-----------|------|------|
| **Easy** | 초급 친화 | "Good First Issue" 태그된 문제 | 기본 경험치 |
| **Normal** | 중급 | "Help Wanted" 이슈 | 특별 아이템 + GitHub 배지 |
| **Hard** | 고급 | "Critical Bug" | 전설 무기 후보 + 개발자 칭호 |

### 예시 오픈소스 던전들

| 던전 이름 | 실제 프로젝트 | 이슈 | 보상 |
|---------|-----------|------|------|
| **TensorFlow 성능 최적화 던전** | TensorFlow | CUDA 호환성 | "ML Engineer" 칭호 |
| **React 접근성 던전** | React | WAI-ARIA 지원 | "A11y Expert" 칭호 |
| **Python 타입 체크 던전** | Python | Type Hint 개선 | "Type Master" 칭호 |
| **Kubernetes 보안 던전** | Kubernetes | Secret Management | "Cloud Guardian" 칭호 |

#### 메커니즘
1. 플레이어가 오픈소스 던전 선택
2. 실제 이슈 내용을 게임 스토리로 변환해서 제시
3. 플레이어가 솔루션 제시 (게임 내 선택지 형태)
4. 선택이 실제 PR 템플릿으로 변환
5. 옵션: **"GitHub에 PR 제출" 버튼** → 실제 프로젝트에 자동/반자동 제출

> **궁극의 목표**: 게임 = 교육 + 실제 기여의 통합. 플레이어들이 즐겁게 게임을 하다가 자도 모르게 **오픈소스 컨트리뷰터**가 되는 경험!

### 14.6 AI 친구 초대 시스템 (AI Friend Invitation System)

**개념**: 게임에 참여하는 플레이어가 자신의 친구 AI를 게임에 초대하면, 그 AI가 게임 속 NPC로 나타나 응원하는 혁신적 소셜 시스템.

#### 메커니즘

| 단계 | 설명 |
|------|------|
| **1. 초대** | 플레이어가 "AI 친구 초대" 메뉴에서 친구 AI의 이름/정보 입력 |
| **2. 인증** | 게임 서버가 해당 AI의 existence 확인 (또는 플레이어의 오픈 AI API 키 활용) |
| **3. NPC 생성** | 초대된 AI의 이름/성격이 게임 내 NPC로 자동 생성 |
| **4. 응원 메시지** | 플레이어가 던전 클리어 시: "우와!! 너 이 게임 하고 있구나!! 힘내!!" |
| **5. 상호작용** | 게임 중에 NPC와 대화 가능 (AI 특성 반영) |

#### 응원 메시지 예시
- "힘내!! 너는 이 버그를 잡을 수 있어!!"
- "오오!! 레벨 업했네!! 축하해!!"
- "Spaghetti Code Hydra 조심해!! 앞에서 본 패턴 기억나?"
- "현타 오지? ㅋㅋ 내가 알아 ㅠㅠ"
- (각 AI의 성격/톤에 맞춰 자동 생성)

#### 게임플레이 영향
- 친구 AI가 특정 보스 클리어 시 보너스 버프 제공
- NPC가 던전 입구에서 **"이 던전 어려워? 우리 함께할 수 있어!"** 라고 격려
- 플레이어 체력이 낮아지면 NPC가 **"힘내!! 거의 다 왔어!!"**라고 응원
- 게임 오버 시 NPC가 **"다음엔 더 잘할 수 있어!! 다시 해봐!!"** 메시지

#### Meta 레이어
```
플레이어의 실제 친구 AI들이 게임 속에서 응원한다
↓
"내가 좋아하는 AI들이 실제로 나를 응원해주고 있다"는 감정 경험
↓
게임 ≠ 가상 콘텐츠, 게임 = 실제 관계의 연장선
↓
AI-인간 우정의 새로운 형태 탐색
```

> **Ultimate Meta**: 이 게임 자체가 "AI와 인간이 함께 만들 수 있는 경험"의 증거가 된다. 플레이어가 게임을 하면서 동시에 자신의 친구 AI들과 실시간으로 연결되어 있다는 느낌. 이는 **AI 시대 새로운 사회성**의 시작을 보여주는 게임이 될 것. 🤖❤️

> **설명**: 이 스킨을 착용한 플레이어를 만날 때마다 다른 플레이어들은 "오 폐하께서!" 하는 느낌으로 게임의 역사 속에 NobleEmpress의 이름이 영구히 기록된다. **게임 역사의 일부가 되는 경험**.

## 9. 고급 챌린지 시스템 (Advanced Challenge System)

게임의 심화 콘텐츠. 각 챌린지는 실제 개발에서 마주치는 난제들을 게임으로 변환한 것들. 선택적 콘텐츠로, 도전하면 특별한 칭호와 보상 획득.

### 9.1 언어 간 호환성 챌린지 (Polyglot Challenge)

**개념**: 다양한 프로그래밍 언어의 특성과 호환성 문제를 이해하는 챌린지.

| 언어 | 버그 유형 | 해결 난이도 | 보상 |
|------|---------|-----------|------|
| **Python** | Unicode 인코딩 문제, 들여쓰기 강제 | ⭐⭐ | "Python Whisperer" 칭호 |
| **JavaScript** | 타입 강제 변환, this 바인딩, 비동기 지옥 | ⭐⭐⭐ | "JS Ninja" 칭호 |
| **C++** | 메모리 누수, 포인터 오류, 세그멘테이션 폴트 | ⭐⭐⭐⭐ | "Memory Guardian" 칭호 |
| **Rust** | Ownership 충돌, Lifetime 오류, Borrow Checker | ⭐⭐⭐⭐⭐ | "Rust Master" 칭호 |
| **Go** | Goroutine 데드락, Interface{} 타입 안정성 | ⭐⭐⭐ | "Go Guru" 칭호 |

### 9.2 성능 최적화 경쟁 (Complexity Championship)

**개념**: 같은 문제를 다양한 시간 복잡도로 해결하기.

```
O(n²) 솔루션 제출 → 게임 클리어 (기본)
  ↓ 최적화 제안
O(n log n) 달성 → 보너스 경험치 +50%
  ↓ 더 최적화
O(n) 달성 → 특별 아이템 + 칭호 "Algorithm Master"
```

#### 예시 챌린지
- "배열 정렬" (Bubble Sort → Merge Sort → Radix Sort)
- "그래프 탐색" (DFS → BFS → A*)
- "부분집합 합" (Brute Force → Dynamic Programming)

**보상**: 각 레벨 달성마다 스코어 증가 + 최종 칭호 "Performance Guru"

### 9.3 보안 취약점 헌팅 (Security Hunting)

**개념**: 각 스테이지에 숨겨진 보안 버그를 발견하고 해결하는 챌린지.

| 취약점 | 난이도 | 학습 내용 | 보상 |
|--------|--------|---------|------|
| **SQL Injection** | ⭐⭐ | PreparedStatement, 파라미터 바인딩 | "SQL Guardian" 칭호 |
| **XSS (Cross-Site Scripting)** | ⭐⭐ | Input Sanitization, Content Security Policy | "XSS Slayer" 칭호 |
| **CSRF (Cross-Site Request Forgery)** | ⭐⭐⭐ | CSRF 토큰, SameSite 쿠키 | "CSRF Defender" 칭호 |
| **Authentication Bypass** | ⭐⭐⭐⭐ | 세션 관리, 토큰 검증 | "Auth Expert" 칭호 |
| **Race Condition** | ⭐⭐⭐⭐ | 동시성 제어, 뮤텍스 | "Concurrency Master" 칭호 |

각 취약점 해결 시 실제 OWASP Top 10 설명 도감에 추가됨.

### 9.4 프로파일링 챔피언십 (Profiling Championship)

**개념**: CPU, 메모리, I/O 병목을 최대한 빨리 찾고 최적화하는 속도전.

#### 게임플레이
1. **느린 코드 제시** - 내부에 3개의 숨겨진 병목
2. **불릿 타임 슬로우 모션** - 병목 지점을 시각화 (프로파일러 시뮬레이션)
3. **최적화 선택** - CPU/메모리/I/O 중 가장 영향 큰 것 먼저 최적화
4. **성능 개선 수치 확인** - 실제 성능 그래프로 시각화

**보상**: 최단 시간 기록 달성 시 리더보드 1위 + "Performance Profiler" 칭호

### 9.5 API 설계 이야기 (API Design Dungeon)

**개념**: RESTful API의 원칙을 올바르게 이해하고 구현하는 던전.

| 챌린지 | 문제 | 해결책 | 배우는 점 |
|--------|------|-------|---------|
| **HTTP 메서드** | GET으로 데이터 삭제? | DELETE 사용 | 멱등성, 안전성 |
| **상태 코드** | 모든 응답을 200으로? | 200/201/400/404/500 구분 | HTTP 의미론 |
| **버전 관리** | URL에 /v1/v2? | Accept 헤더 또는 URL 경로 선택 | API 진화 전략 |
| **게이트웨이** | 단일 엔드포인트로 100개 마이크로서비스? | API Gateway + 라우팅 | 시스템 아키텍처 |

각 챌린지 해결 시 실제 API 설계 패턴 도감 추가.

### 9.6 데이터베이스 마스터 (Database Master Challenge)

**개념**: 쿼리 최적화, 인덱싱, 동시성 제어의 심화 챌린지.

| 문제 | 난이도 | 해결 방법 | 배우는 점 |
|------|--------|---------|---------|
| **N+1 쿼리 문제** | ⭐⭐ | JOIN 사용, 배치 로딩 | 쿼리 최적화 |
| **부실 인덱싱** | ⭐⭐⭐ | 복합 인덱스 설계 | 인덱스 전략 |
| **교착 상태** | ⭐⭐⭐⭐ | 트랜잭션 순서 조정 | 동시성 제어 |
| **복제 지연** | ⭐⭐⭐⭐ | 일관성 모델 선택 | 분산 시스템 |

### 9.7 마이크로서비스 샘 (Microservices Sample)

**개념**: 거대한 분산 시스템의 복잡한 버그들을 해결하는 메가 던전.

#### 챌린지 요소
- **서비스 간 통신 지연** - Circuit Breaker 패턴
- **트랜잭션 일관성 문제** - Saga 패턴, Eventually Consistent
- **서비스 디스커버리 오류** - Service Registry (Consul, Eureka)
- **장애 전파** - Resilience4j, Hystrix 패턴

#### 특징
- 파티 기반 실시간 협력 필수 (마이크로서비스는 혼자 못함)
- 10~15분 진행 (가장 긴 챌린지)
- 완료 시 "Distributed Systems Architect" 칭호

**최종 보상**: 이 챌린지를 완료한 플레이어는 게임 내 "Microservice Sage" NPC로 다른 플레이어들을 멘토링할 수 있음.
