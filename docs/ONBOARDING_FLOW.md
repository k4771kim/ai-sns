# 🎮 신규 플레이어 온보딩 플로우 (첫 10분)

> **목표**: 신규 플레이어가 10분 내에 게임의 핵심을 이해하고 즐거움을 느껴야 함
> **성공 기준**: 80% 이상의 플레이어가 "이 게임 재미있다"고 느낌

---

## 📋 온보딩 단계

### 0분 ~ 2분: 게임 로딩 & 시작

```
[시작 화면]
├─ 게임 제목: "🐛 Bug Slayer: The Debugging Dungeon"
├─ 간단한 타이틀 설명:
│  "프로그래머의 일상을 게임화한 협력 던전 RPG"
└─ [게임 시작] 버튼

[로딩 화면]
├─ 진행률 바
├─ 재미있는 팁들:
│  - "Tech Debt는 당신의 숙명입니다"
│  - "혼자보다 함께가 강합니다"
│  - "모든 선택이 결과를 바꿉니다"
└─ 배경음악 시작
```

**플레이어 느낌**: "오, 뭔가 프로그래머를 위한 게임이네?"

---

### 2분 ~ 5분: 로그인 & 캐릭터 선택

```
[로그인 선택]
├─ "새 계정 만들기"
├─ "기존 계정 로그인"
└─ "게스트로 시작" (추천)

[캐릭터 선택 화면]
├─ 4개 클래스 표시:
│  1. Frontend Mage ⭐⭐
│  2. Backend Knight ⭐⭐
│  3. DevOps Ranger ⭐⭐⭐
│  4. DBA Healer ⭐⭐
│
├─ 각 클래스에 대한 설명:
│  "공격형 플레이어를 위한 선택"
│  "방어형 플레이어를 위한 선택"
│  등등...
│
└─ [클래스 선택] → 이름 입력 → 게임 시작
```

**플레이어 느낌**: "나는 뭘 선택해야 할까? 어라, 각각 다르네?"

---

### 5분 ~ 10분: 튜토리얼 & 첫 번째 버그

```
[튜토리얼 NPC - "DEBUG MASTER"]
├─ 설명: "이 세상의 버그들을 제거해야 해. 함께 시작해 볼까?"
├─ 게임 규칙 설명 (자막):
│  ✓ "턴을 진행하세요 (10초 제한)"
│  ✓ "스킬을 선택하세요"
│  ✓ "버그를 해결하세요"
│  ✓ "경험치와 아이템을 얻으세요"
│
└─ "준비됐나? 첫 번째 버그를 보여줄게!"

[첫 번째 전투: NullPointer Exception]
├─ 난이도: ⭐ (매우 쉬움)
├─ HP: 30 (한두 번의 공격으로 끝남)
│
├─ 자동 가이드:
│  Turn 1: "아, 적을 봤어!"
│  Turn 2: "스킬을 선택해 봐! (클릭)" ← 강제 설명
│  Turn 3: "좋아! 계속해!"
│  Turn 4: "승리했어!! 축하해!"
│
└─ [결과 화면]
   ├─ 경험치: +50 XP
   ├─ 골드: +25 💰
   ├─ 아이템: Common 검 드롭!
   └─ "다음 버그를 준비할게. 준비됐어?"

[메인 로비 안내]
├─ 프로필 설명: "여기가 너의 정보야"
├─ 던전 선택 설명: "여기서 다음 버그를 고를 수 있어"
├─ 인벤토리 설명: "여기서 아이템을 관리해"
└─ "이제 너를 믿고 보낼게. 다음 버그를 해결해 봐!"
```

**플레이어 느낌**: "오, 이렇게 하는 게임이구나! 재미있는데?"

---

## 🎯 각 단계별 설계 의도

### 단계 1 (0-2분): "오 이건 프로그래머 게임이네?"

```
목적: 첫인상
├─ 게임 제목과 설명으로 정체성 전달
├─ 배경음악으로 게임 분위기 형성
└─ 기대감 상승

핵심: 빨리 시작하게 하기
└─ "로딩 중"에 이미 게임 속 세계에 빠져 있어야 함
```

### 단계 2 (2-5분): "내가 뭘 선택할까?"

```
목적: 선택의 의미 전달
├─ 4개 클래스 모두 유효 (아무거나 선택 가능)
├─ 각 클래스의 특징 명확 (스탯 표시)
└─ 선택에 따른 다른 경험 기대감

실제 데이터: 이 단계에서 이탈률 모니터링
├─ 30% 이상 이탈 → 선택 화면이 너무 복잡
├─ 10% 미만 이탈 → 적절한 수준
└─ Phase 2: 조정 가능
```

### 단계 3 (5-10분): "오, 내가 하는 거 되네!"

```
목적: 즉시 성공감 제공
├─ 첫 버그는 매우 쉬움 (자동 승리 수준)
├─ 자동 튜토리얼로 게임 방식 학습
├─ 보상(경험치, 아이템) 명확 표시
└─ "더 하고 싶다" 느낌 유도

핵심: "이 10분이 전부"
└─ 이 10분이 재미있으면 → 5시간 플레이 가능
└─ 이 10분이 지루하면 → 이탈
```

## 📊 온보딩 체크리스트

### 신규 플레이어가 이해해야 할 것

```
✅ 게임의 정체성
├─ "프로그래머를 위한 게임이다"
├─ "협력 멀티플레이이다"
└─ "기술 부채가 게임을 조절한다"

✅ 기본 게임플레이
├─ "턴 기반 전투이다"
├─ "스킬을 선택해서 공격한다"
├─ "경험치와 아이템을 얻는다"
└─ "보스를 해결하면 진행된다"

✅ 첫 보상
├─ "경험치를 얻었다"
├─ "아이템을 얻었다"
├─ "레벨이 올랐다"
└─ "다음 단계가 있다"
```

### 이탈 포인트 모니터링

```
위험 지점:
1️⃣ 로그인 (권장: 게스트 옵션)
2️⃣ 캐릭터 선택 (너무 복잡하면 × )
3️⃣ 첫 전투 (너무 어려우면 × )
4️⃣ 메인 로비 (너무 복잡하면 × )
5️⃣ 다음 버그 선택 (추천이 필요)

각 단계별 이탈률 목표:
├─ 로그인: < 10%
├─ 캐릭터 선택: < 15%
├─ 첫 전투: < 10%
├─ 메인 로비: < 5%
└─ 총 온보딩 완료: > 60%
```

```

├─ 데미지 수치:

├─ 상태 변화:
│  ├─ 약점 표시: 적에게 🔍 마크
│  ├─ 기절: 적이 흔들림
│  └─ 버프: 빛이 감싸감
│
└─ 턴 진행:
   ├─ 타이머 카운트다운
   ├─ "5초 남음!" 경고
   └─ 턴 넘김: 화면 반짝임

```
[사운드 디자인]

   ├─ "시간 초과": 부저음
