## 2. 핵심 시스템: Tech Debt

### 💀 Tech Debt란?
**버그를 해결하지 않고 계속 모험하면 쌓이는 "기술 부채"**

게임의 **난이도 조절 기제**이자 **엔딩 분기점**입니다.

### 📊 Tech Debt 메커니즘

#### 증가 방식
```
매 턴마다 자동으로 증가:
- 기본: +1 (매 턴)
- 버그를 제때 해결하지 않으면: +1~3
- 어려운 전투에서 회피: +2~5

범위: 0 ~ 100
```

#### Tech Debt 레벨별 효과

| Tech Debt | 상태 | 플레이 영향 | 플레이어 느낌 |
|-----------|------|-----------|------------|
| 0-20 | 청결 | 적 공격력 -20% | "여유로움" |
| 21-50 | 경고 | 적 공격력 +0% (정상) | "괜찮음" |
| 51-80 | 위험 | 적 공격력 +30% | "긴장됨" |
| 81-100 | 위기 | 적 공격력 +50% + 추가 미니 보스 | "절박함" |

#### 엔딩 분기 (Tech Debt로 결정)

```
Tech Debt < 40:
├─ Good Ending ✨
├─ "완벽한 코드로 세상을 구했다!"
└─ 보너스 콘텐츠 해제

Tech Debt 40-70:
├─ Normal Ending
├─ "그래도 버그는 해결했다"
└─ 일반 엔딩

Tech Debt > 70:
├─ Bad Ending 💔
├─ "아, 기술 부채가 너무 많았다..."
└─ 다음 회차 도전권
```

#### 전략적 선택
```
플레이어의 선택지:
- "빠른 클리어" vs "완벽한 클리어"
- "어려운 버그는 나중에" vs "지금 바로"
- "속도 플레이" vs "여유 플레이"

각 선택이 다른 엔딩으로 이어짐 → 재플레이 유도
```

## 4. 게임플레이 루프

### 🎮 기본 턴 기반 전투

```
1. 플레이어 턴
   ├─ 스킬 선택
   ├─ 타겟 선택
   └─ 실행

2. NPC/파티원 턴 (멀티플레이시)
   ├─ AI 또는 플레이어 선택
   └─ 실행

3. 적 턴
   ├─ 적 행동
   └─ 플레이어에게 데미지

4. 턴 종료
   ├─ Tech Debt +1
   ├─ 상태이상 업데이트
   └─ 전투 계속 여부 확인
```

### 🏆 전투 결과

```
버그 해결 (승리):
├─ 경험치 획득 (100~500)
├─ 골드 획득 (50~200)
├─ 드롭 아이템 (Random)
└─ Tech Debt 변화 (맥락에 따라)

패배:
├─ 게임 오버 또는 회귀
└─ Tech Debt 증가 (큼)
```

- "곡예사" - 1번만 피하고 전체 챕터 클리어
